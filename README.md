# iPad用「電卓」PWA（広告なし）

あなたが欲しい「アイコン化したらアプリっぽく動く電卓」は、ネイティブアプリじゃなく **PWA** が現実解です。

ただし事実として、**iPadでPWAを“アプリ化（ホーム画面追加＋オフライン）”するにはHTTPS配信がほぼ必須**です。  
`file://` でHTMLを直開きしたり、同一LANの `http://<MacのIP>:8000` みたいな配信だと、iOSは `service worker` が動かずオフライン化が死にます。

## できること
- **広告なし**
- **軽い**（依存ゼロのHTML/CSS/JS）
- 基本演算: `+ - * /`
- 小数、符号反転、バックスペース、AC、%（入力値を100で割る）
- キーボード対応（外付けキーボードでも可）
- PWA（ホーム画面追加・スタンドアロン表示・オフラインキャッシュ）

## ファイル
- `index.html` UI
- `styles.css` 見た目
- `app.js` ロジック（`eval`不使用）
- `manifest.webmanifest` PWA設定
- `sw.js` オフライン用キャッシュ
- `icon.svg` アイコン（SVG）

## iPadに入れる（現実的な手順）
1. どこかHTTPSでこのフォルダを静的ホスティングする（GitHub Pages等）
2. iPadのSafariで開く
3. 共有 → **ホーム画面に追加**

※「ホスティング無しでアプリ化」は無理筋です。ここをケチると、結局“ただのブックマーク”になります。

## アイコンをちゃんとしたPNGにしたい場合（任意）
iOSのホーム画面アイコンはPNG推奨です。最低でも `icon-180.png`（apple-touch-icon）を作ると見栄えが安定します。

1. `icon.svg` をMacで開く（プレビュー等）
2. 1024x1024のPNGとして書き出す
3. 必要サイズにリサイズして同フォルダに置く（例: `icon-180.png`）
4. `index.html` の `apple-touch-icon` 行を有効化

（PNGを追加したら `manifest.webmanifest` の `icons` にPNGも足すとより確実です）
